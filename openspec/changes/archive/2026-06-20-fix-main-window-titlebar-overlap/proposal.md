# Change：修复主窗口标题栏内容重叠

## 为什么需要此变更

主窗口恢复到非首个侧栏目的地时，原生侧栏可能自动滚动，并将顶部“总览”行放到统一工具栏下方。该行会与窗口红黄绿按钮和侧栏切换按钮重叠，标题与说明也会被裁切，影响可读性和点击安全。

## 变更内容

- 让侧栏滚动内容始终避开主窗口标题栏与窗口控制按钮占用的安全区域。
- 安全区间距根据实际窗口环境计算，不写死某个 macOS 版本的标题栏高度。
- 保持原生 `NavigationSplitView`、`.sidebar` 列表、选中态、滚动与导航恢复行为。
- 增加纯值布局计算测试，覆盖零安全区和已有安全区两种情况。

## 影响范围

### 修改的现有能力

- `main-dashboard`：修正主窗口侧栏在统一工具栏下的顶部布局。

## 非目标

- 不重设计 Dashboard 或功能占位页。
- 不改变 Launcher 面板、业务操作或权限策略。
- 不用 AppKit 改写主窗口或手动移动红黄绿按钮。

## 成功标准

- 主窗口启动和恢复任意目的地时，侧栏首行不与窗口按钮、标题或工具栏重叠。
- 侧栏仍可正常选择、滚动和折叠。
- 工程可编译且全部自动化测试通过。
