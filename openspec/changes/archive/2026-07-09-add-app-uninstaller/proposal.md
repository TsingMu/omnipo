# Change：添加 App Uninstaller 核心能力

## 为什么需要此变更

Omnipo 目前侧边栏中已有 Uninstaller 入口，但页面仍是占位状态。用户可以看到“应用卸载”的入口，却无法列出已安装应用、查看应用占用空间、执行卸载，或清理应用卸载后遗留的缓存与配置文件。

本 change 将 Uninstaller 从占位页升级为可用能力。首版必须支持两种卸载意图：

- 普通卸载：删除应用本体。
- 完全删除：在用户授权且系统允许的范围内，删除应用本体，并同时删除用户选中的、可安全归属的缓存、Application Support、Preferences、Logs、Saved Application State、Containers 等关联文件。

由于删除操作不可轻率执行，本 change 的核心不仅是“能删”，还包括“先发现、可预览、可选择、可确认、可降级、可追踪”。任何归属不明确、共享风险高、权限不足或系统保护的数据都必须明确展示风险与原因，不得静默删除。

## 变更内容

- 新增 `app-uninstaller` 能力规范、模型、服务协议和系统实现边界。
- 用真实 App Uninstaller 页面替换当前占位页。
- 枚举本机可见应用，展示应用名称、bundle identifier、路径、图标、大小与来源位置。
- 支持用户选择一个应用并发起卸载预览。
- 提供卸载模式选择：
  - 仅卸载应用本体。
  - 完全删除应用及关联文件。
- 完全删除模式必须扫描并展示当前权限范围内可发现的可归属关联文件，包括但不限于：
  - `~/Library/Caches/<bundle-id 或相关名称>`
  - `~/Library/Application Support/<app 或 bundle-id>`
  - `~/Library/Preferences/<bundle-id>.plist`
  - `~/Library/Logs/<app 或 bundle-id>`
  - `~/Library/Saved Application State/<bundle-id>.savedState`
  - `~/Library/Containers/<bundle-id>`
  - `~/Library/Group Containers` 中可安全确认归属的条目
- 完全删除预览必须按文件类别分组展示，并为每个类别注明删除后果，例如缓存会被重建、偏好设置会被重置、Application Support 可能包含本地数据。
- 用户必须能在完全删除预览中逐项选择或取消选择关联文件。
- 删除前必须显示二次确认，清楚列出将被移除的应用和关联文件数量/大小。
- 删除执行优先移动到废纸篓；无法移动到废纸篓时必须明确提示失败原因，首版不提供应用内永久删除。
- 删除过程必须产生本地操作结果：成功、部分成功、失败、跳过和不可访问原因。
- 日志不得包含用户路径、文件名或隐私数据，只允许稳定错误码和汇总计数。

## 影响范围

### 新增能力

- `app-uninstaller`：应用发现、卸载预览、普通卸载、完全删除、关联文件扫描、删除执行结果展示。

### 修改的现有能力

- `application-foundation`：新增 `UninstallerService` 依赖装配和应用级状态入口；若采用 Finder 删除路径，需要新增 Apple Events entitlement，并在 Info.plist 增加控制 Finder 的用途说明。关联文件扫描优先使用用户授权目录与 security-scoped bookmark；Full Disk Access 仅作为受保护路径读取的补充引导，不承诺解除全部文件系统或沙箱限制。
- `main-dashboard` / `launcher`：既有 Uninstaller 导航目标从占位能力升级为真实功能页。
- `disk-analysis` / `cleaner`：可复用文件大小统计、权限降级、删除确认和操作日志模式，但不把 Uninstaller 合并到 Cleaner。

## 非目标

- 不删除系统应用、Apple 受保护应用或位于系统保护路径中的应用。
- 不绕过 SIP、TCC、沙盒或文件系统权限。
- 不默认删除共享 Group Containers，除非能够安全确认归属并由用户明确勾选。
- 不解析应用私有数据库内容，不读取用户文档、聊天记录、钥匙串或浏览器隐私数据。
- 不在首版实现批量卸载、自动推荐卸载、后台定时清理或远程规则库。
- 不实现内核扩展、登录项、LaunchAgent/LaunchDaemon 的强制卸载；首版只展示可发现项并保守跳过高风险项。
- 不上传应用列表、路径、文件名、缓存内容或删除结果。

## 用户价值

- 用户可以在 Omnipo 内查看本机应用并执行卸载。
- 用户可以自行选择普通卸载或完全删除，避免卸载后残留大量缓存。
- 用户在完全删除前可以看到关联文件明细，并控制哪些文件会被删除。
- 用户可以理解哪些文件因为权限、系统保护或共享风险而无法删除。
- 删除操作有明确确认与结果反馈，降低误删风险。

## 风险概述

- 删除应用和关联文件存在误删风险，尤其是共享容器、插件、用户文档和跨应用缓存。
- 应用归属判断可能不完美，不能仅靠名称模糊匹配自动删除高风险文件。
- macOS 权限、沙盒、SIP 和文件锁定会导致部分文件无法删除。
- 某些应用正在运行时，删除可能失败或留下残留。
- 文件路径和文件名属于隐私信息，日志必须严格脱敏。
- 删除依赖用户授权路径或 Finder automation（控制 Finder），关联文件扫描可能受 Full Disk Access、沙箱与文件系统权限影响；授权拒绝或撤销时必须明确降级而非静默失败。

详细风险、删除策略和降级方案见 `design.md`。

## 成功标准

- Uninstaller 页面不再显示占位文案，能够列出本机可见应用。
- 用户可以搜索/筛选应用并查看应用基本信息和图标。
- 用户选择普通卸载时，预览和确认只包含应用本体。
- 用户选择完全删除时，预览必须包含应用本体和可安全归属的缓存、配置、支持文件等关联文件。
- 完全删除预览必须按类别展示，并注明每类文件删除后的影响或后果。
- 用户可以在完全删除预览中逐项取消关联文件。
- 删除前必须二次确认，确认内容区分普通卸载与完全删除。
- 删除执行优先移动到废纸篓；首版不提供应用内永久删除。
- 删除完成后显示成功、部分成功、失败和跳过原因。
- 权限不足、系统保护、归属不明确或共享风险必须展示为不可删除/需确认状态，不得静默删除。
- 日志不包含用户路径、文件名、应用私有数据或关联文件明细。
- 工程持续可编译，核心模型、扫描、预览、删除执行和降级路径均有测试覆盖。
