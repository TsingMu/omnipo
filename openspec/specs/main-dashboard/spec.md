## Requirements
### Requirement: 主窗口必须提供清晰的分组导航

系统 MUST 为八个稳定目的地提供位置稳定的分组侧栏，按概览、效率工具和系统工具展示中文标题、语义图标与简短说明，并为 Dashboard 和全部功能详情页提供统一的标题栏安全边界。选择变化不得导致原本可见的侧栏内容整体上下位移，任何详情内容不得进入窗口工具栏区域；入口选择只导航，不得启动敏感业务操作。

#### Scenario: 连续点击全部入口

- **当** 用户依次点击总览、快速启动、剪切板、磁盘清理、应用卸载、权限审计、微信管理和系统监控
- **那么** 未发生手动滚动时侧栏分组与入口保持固定坐标
- **并且** 只有选中高亮和详情内容发生变化
- **并且** 当前入口仍具有明确的选中视觉和辅助功能状态

#### Scenario: 显示任一详情页

- **当** 主窗口显示任一稳定目的地
- **那么** 页面标题、图标、说明、卡片和操作均完整位于工具栏下方
- **并且** 页面内容不会被标题栏裁切

#### Scenario: 使用键盘或小窗口导航

- **当** 用户使用方向键切换入口，或窗口高度不足以显示全部入口
- **那么** 选择按稳定目的地顺序移动并在首尾停止
- **并且** 侧栏可以滚动访问全部入口
- **并且** 仅键盘导航到不可见目标时允许按需滚动

### Requirement: Dashboard 必须说明产品定位与当前状态

Dashboard MUST 展示 Omnipo 品牌说明、启动磁盘状态卡和安全说明。未接入真实磁盘服务时，系统必须明确显示尚未扫描，不得展示虚构容量或占用率。

#### Scenario: 首次查看 Dashboard

- **假如** 当前没有真实磁盘扫描结果
- **当** 用户打开 Dashboard
- **那么** 状态卡显示“尚未扫描”
- **并且** 不读取用户文件或隐私数据
- **并且** 不展示演示容量、占用百分比或虚构节省空间

### Requirement: Dashboard 必须提供安全的快捷导航

Dashboard MUST 提供磁盘扫描、应用卸载、权限审计和微信管理四个快捷入口。这些入口只能导航到对应页面，不得自动开始扫描、删除、审计或读取微信数据。

#### Scenario: 使用快捷入口

- **当** 用户点击任一 Dashboard 快捷入口
- **那么** 主窗口选择对应侧栏目的地
- **并且** 不自动执行该页面的业务操作

### Requirement: Dashboard 必须适配 macOS 外观与窗口尺寸

Dashboard MUST 使用系统语义色和材质，并在应用最小窗口尺寸下保持内容可读和可访问。

#### Scenario: 缩小窗口或切换外观

- **当** 用户将窗口缩小到支持的最小尺寸或切换浅色与深色外观
- **那么** 内容可通过自适应布局或滚动继续访问
- **并且** 文字、卡片与操作保持足够对比度
