# launcher 增量规范

## MODIFIED Requirements

### Requirement: Launcher 必须提供独立搜索面板

系统必须（MUST）提供适合快速唤起的独立搜索面板；同时，主窗口中的“快速启动”页面 MUST 提供等价的内嵌 Launcher 工作台，允许用户不离开主窗口直接完成查询、选择与执行。

#### Scenario: 打开主窗口快速启动页

- **当** 用户在主窗口选择“快速启动”
- **那么** 页面显示 Launcher 搜索框与结果区域
- **并且** 不再显示 “Phase 0 暂未实现” 占位说明
- **并且** 用户无需额外打开浮动面板也可开始搜索

#### Scenario: 空查询查看快速入口命令

- **当** 用户打开主窗口“快速启动”页
- **并且** 当前查询为空
- **那么** 页面显示六个内置功能命令
- **并且** 行为与独立 Launcher 面板中的空查询一致

#### Scenario: 在主窗口页执行结果

- **当** 用户在主窗口“快速启动”页点击结果或按 Return 执行选中结果
- **那么** 系统执行与独立 Launcher 面板相同的结果执行逻辑
- **并且** 页面本身保持可见
- **并且** 不因执行而自动触发扫描、删除或额外权限请求

#### Scenario: 主窗口页执行失败

- **当** 主窗口“快速启动”页中的结果执行失败
- **那么** 页面显示安全、可理解的失败提示
- **并且** 不暴露文件路径、用户目录名或其他敏感信息
