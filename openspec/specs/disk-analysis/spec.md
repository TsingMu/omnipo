# disk-analysis Specification

## 元信息

- **能力名称**:disk-analysis
- **首次实现 change**:`add-disk-analysis`
- **验收日期**:2026-06-24
- **当前状态**:已实现并合并

## 概述

disk-analysis capability 提供启动卷容量摘要与大文件列表的只读视图。容量摘要来自系统卷级元数据;大文件列表由用户通过 `NSOpenPanel` 主动授权扫描根目录后,基于 security-scoped bookmark 持久化,扫描只读 `fileSize`/`contentModificationDate`/`isRegularFile` 元数据,不读取文件内容、不建立自有索引、不执行删除或清理动作。

## Requirements

### Requirement: 磁盘分析页必须提供只读容量概览

磁盘分析页 MUST 展示当前启动卷或主工作卷的只读容量摘要，包括已用空间、可用空间和总容量。此能力只允许读取卷级元数据，不得通过递归遍历目录、建立自有全盘索引或读取文件内容来生成首页或详情页的容量数字。

#### Scenario: 容量摘要可用

- **当** 用户打开磁盘分析页
- **并且** 系统已成功读取目标卷容量摘要
- **那么** 页面显示已用空间、可用空间和总容量
- **并且** 与 Dashboard 状态卡展示相同来源的数据

#### Scenario: 容量摘要不可用

- **当** 用户打开磁盘分析页
- **并且** 系统暂时无法读取目标卷容量摘要
- **那么** 页面显示容量信息不可用及原因
- **并且** 不以 0 B 或空状态伪装成功结果

### Requirement: 磁盘分析页必须允许显式刷新

磁盘分析页 MUST 提供用户可感知的显式刷新入口。刷新只能重新读取卷级容量摘要，不得因此自动启动目录扫描、清理建议计算或任何删除动作。

#### Scenario: 用户手动刷新

- **当** 用户在磁盘分析页触发刷新
- **那么** 系统重新读取目标卷容量摘要
- **并且** Dashboard 与磁盘分析页共享更新后的状态
- **并且** 不开始目录分析或清理

### Requirement: 磁盘分析页必须展示大文件列表

磁盘分析页 MUST 提供大文件展示区块，并按文件大小从大到小排序结果。每条结果至少展示文件名、路径和大小等只读元信息；系统不得为此读取文件内容。在 App Sandbox 下，默认用户目录不可读，必须由用户通过 `NSOpenPanel` 显式授权扫描根目录;授权以 security-scoped bookmark 形式持久化。

#### Scenario: 大文件结果可用

- **当** 用户已授权扫描目录
- **并且** 系统已成功生成大文件结果
- **那么** 页面按大小降序展示文件列表
- **并且** 每条结果包含文件名、路径和大小
- **并且** 不展示文件内容预览

#### Scenario: 大文件结果不可用

- **当** 用户尚未授权扫描目录,或扫描失败
- **那么** 页面显示大文件列表不可用及原因
- **并且** 不伪造示例文件结果

#### Scenario: 跨重启保留授权

- **当** 用户重启应用
- **并且** 之前已授权扫描目录
- **那么** 系统从持久化的 security-scoped bookmark 还原授权
- **并且** 不要求用户再次选择

### Requirement: 大文件刷新必须保持保守边界

大文件刷新 MUST 只读取文件系统元数据并支持结果条数限制与旧任务取消。系统不得在刷新过程中执行删除、移动到废纸篓、文件内容解析或自有持久索引构建。

#### Scenario: 用户刷新大文件列表

- **当** 用户在磁盘分析页触发刷新
- **那么** 系统重新读取大文件元数据
- **并且** 结果数量被限制在预设上限内(默认 50)
- **并且** 若上一次刷新仍在进行，则旧任务被取消或废弃
- **并且** 不执行删除或内容读取

#### Scenario: 扫描系统子树

- **当** 授权目录下包含 `Library`、`.Trash`、`Caches`、`Containers`、`node_modules`、`.git` 等系统/缓存子树
- **那么** 扫描跳过这些子树及其全部后代
- **并且** 不把系统缓存或开发依赖展示为用户的大文件

### Requirement: 磁盘分析页必须清晰标注未实现能力

在目录分析、分类占用和清理建议尚未实现前，磁盘分析页 MUST 明确说明这些能力尚未提供，避免用户将当前页面误解为已完成的全量清理工具。

#### Scenario: 查看 Phase 0 说明

- **当** 用户查看磁盘分析页
- **那么** 页面明确说明当前阶段仅提供容量概览
- **并且** 将目录分析、分类占用和清理建议标记为后续能力

### Requirement: 磁盘分析必须保持保守权限边界

磁盘分析 capability MUST 在无额外隐私权限前提下工作。系统不得为了展示容量摘要申请完全磁盘访问、辅助功能、输入监控或其他与卷级容量读取无关的权限。大文件扫描需要的目录访问权限只能由用户主动通过 `NSOpenPanel` 授权,且范围严格限制在所选目录。

#### Scenario: 首次使用磁盘分析

- **当** 用户首次打开 Dashboard 或磁盘分析页
- **那么** 系统不请求额外隐私权限(容量摘要)
- **并且** 不读取文件内容
- **并且** 不产生删除、副作用写入或后台常驻扫描

#### Scenario: 用户撤销授权

- **当** 用户点击"清除"释放大文件扫描授权
- **那么** 系统释放 security scope 并删除持久化 bookmark
- **并且** 不再扫描该目录
