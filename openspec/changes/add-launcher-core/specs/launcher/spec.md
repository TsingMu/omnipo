# launcher 增量规范

## ADDED Requirements

### Requirement：系统必须提供默认全局快捷键

Launcher 必须默认使用 Option + Space 作为全局快捷键。快捷键必须通过只注册明确组合的系统机制实现，不得为了此能力监听全部键盘输入或要求辅助功能、输入监控权限。

#### Scenario：首次启动且默认快捷键可用

- **当** 用户首次启动 Omnipo
- **并且** Option + Space 未被系统或其他应用占用
- **那么** 系统注册 Option + Space
- **并且** 用户可从其他普通前台应用唤起 Launcher
- **并且** 系统不请求辅助功能或输入监控权限

#### Scenario：默认快捷键被占用

- **当** 应用无法注册 Option + Space
- **那么** 应用保持主窗口功能可用
- **并且** 在 Launcher 或设置中提示快捷键冲突
- **并且** 引导用户配置其他组合
- **并且** 不通过扩大键盘监听权限绕过冲突

### Requirement：用户必须能够配置全局快捷键

系统必须在独立设置窗口中显示当前有效快捷键，并允许用户录制、取消和恢复默认快捷键。只有系统成功注册候选组合后，系统才能将其保存为当前配置。

#### Scenario：成功更换快捷键

- **当** 用户录制一个有效且未冲突的快捷键
- **那么** 系统注册新快捷键
- **并且** 注销旧快捷键
- **并且** 仅在本地保存新配置
- **并且** 设置界面显示新快捷键

#### Scenario：候选快捷键冲突

- **当** 用户录制的快捷键被系统或其他应用占用
- **那么** 系统显示明确冲突原因
- **并且** 不保存候选配置
- **并且** 上一次有效快捷键继续工作

#### Scenario：候选快捷键无效

- **当** 用户录制仅修饰键、无修饰键或不受支持的组合
- **那么** 系统拒绝该组合
- **并且** 说明有效组合要求
- **并且** 当前快捷键保持不变

#### Scenario：恢复默认快捷键

- **当** 用户选择恢复默认值
- **并且** Option + Space 可以注册
- **那么** 系统将当前快捷键替换为 Option + Space
- **并且** 在本地保存默认配置

#### Scenario：读取到损坏配置

- **当** 应用启动时读取到损坏或不支持的快捷键设置
- **那么** 应用不崩溃
- **并且** 尝试回退到 Option + Space
- **并且** 日志只记录稳定错误代码，不记录原始设置值

### Requirement：Launcher 必须提供独立搜索面板

系统必须提供适合快速唤起的独立搜索面板。面板必须支持显示、隐藏、重复快捷键切换、当前屏幕定位和搜索输入焦点管理。

#### Scenario：通过快捷键打开面板

- **当** Launcher 面板处于隐藏状态
- **并且** 用户按下当前有效快捷键
- **那么** 系统在当前相关屏幕的可见区域显示面板
- **并且** 搜索框立即获得输入焦点
- **并且** 用户可以直接输入查询

#### Scenario：重复快捷键关闭面板

- **当** Launcher 面板已经显示
- **并且** 用户再次按下当前有效快捷键
- **那么** 系统关闭面板
- **并且** 取消不再需要的搜索任务

#### Scenario：面板失去焦点

- **当** 用户切换到其他窗口或点击面板之外
- **那么** 系统关闭 Launcher 面板
- **并且** 不持续拦截键盘输入

#### Scenario：跨显示器唤起

- **当** 用户在不同显示器或 Space 中唤起 Launcher
- **那么** 面板显示在与当前交互上下文相关的屏幕可见区域
- **并且** 不永久遮挡其他应用

### Requirement：Launcher 必须支持键盘优先操作

用户必须能够不使用鼠标完成查询、选择、执行和关闭流程。

#### Scenario：移动结果选择

- **当** 搜索结果列表存在多个可执行结果
- **并且** 用户按 Up 或 Down
- **那么** 系统移动当前选择
- **并且** 选中结果保持可见

#### Scenario：执行选中结果

- **当** 当前存在可执行选择
- **并且** 用户按 Return
- **那么** 系统执行该结果
- **并且** 默认关闭 Launcher 面板

#### Scenario：取消 Launcher

- **当** Launcher 面板显示
- **并且** 用户按 Escape
- **那么** 系统关闭面板
- **并且** 清理瞬态查询与结果
- **并且** 不执行任何结果

#### Scenario：使用鼠标执行结果

- **当** 用户点击一个可执行搜索结果
- **那么** 系统执行该结果
- **并且** 行为与 Return 执行一致

### Requirement：搜索结果必须包含功能命令

Launcher 必须提供打开剪切板、扫描磁盘、卸载应用、权限审计、查看微信占用和打开系统监控六个内置命令。

#### Scenario：空查询显示命令

- **当** Launcher 面板打开且查询为空
- **那么** 系统显示六个内置功能命令
- **并且** 不启动无界文件搜索

#### Scenario：按关键词搜索命令

- **当** 用户输入与某个命令标题或关键词匹配的文本
- **那么** 系统返回对应命令
- **并且** 完全匹配或前缀匹配优先于弱匹配

#### Scenario：执行打开剪切板命令

- **当** 用户执行“打开剪切板”
- **那么** 系统打开或前置 Omnipo 主窗口
- **并且** 导航到 Clipboard 页面
- **并且** 不在本 change 中读取剪切板历史

#### Scenario：执行其他功能命令

- **当** 用户执行扫描磁盘、卸载应用、权限审计、查看微信占用或打开系统监控
- **那么** 系统导航到对应现有页面
- **并且** 不自动开始扫描、删除、审计或高频监控

### Requirement：搜索结果必须包含应用

Launcher 必须使用系统公开能力发现和匹配可启动应用，并允许用户通过结果启动应用。应用索引必须保留显示名称、Bundle 名称、可执行文件名、Bundle Identifier 以及可安全生成的中文拼音别名，不得因本地化显示名称而丢弃其他可搜索名称。

#### Scenario：搜索已安装应用

- **当** 用户输入与已发现应用名称匹配的查询
- **那么** 系统显示应用结果
- **并且** 结果明确标识为应用类型
- **并且** 重复应用结果被稳定去重

#### Scenario：通过英文名称搜索本地化应用

- **假如** 一个应用的本地化显示名称为“微信”，且 Bundle 名称或可执行文件名为 `WeChat`
- **当** 有效查询为 `wechat`
- **那么** 系统显示微信应用结果
- **并且** 结果标题仍可使用本地化显示名称

#### Scenario：通过中文拼音搜索应用

- **假如** 已发现应用的显示名称为“微信”
- **当** 用户输入 `weixin`、`wei xin` 或 `wx`
- **那么** 系统可通过预计算的全拼、紧凑全拼或首字母别名显示微信应用结果
- **并且** 系统不在每次按键时重复转换全部应用名称

#### Scenario：启动应用

- **当** 用户执行一个有效应用结果
- **那么** 系统通过 macOS 公开工作区 API 启动或激活该应用
- **并且** 关闭 Launcher 面板

#### Scenario：结果行重绘复用应用资源

- **假如** 一个应用的 URL 与图标已经解析并进入进程内缓存
- **当** SwiftUI 因选择或增量结果更新而重复绘制该应用结果行
- **那么** 系统复用已缓存的 URL 与图标
- **并且** 不在每次 `body` 计算时同步重复查询 `NSWorkspace`

#### Scenario：工作区应用发生变化

- **当** 工作区通知表明文件操作、卷挂载或卸载可能改变可用应用
- **那么** 系统使应用 URL 与图标缓存失效
- **并且** 在后台触发应用索引 single-flight 刷新
- **并且** 已显示的 Launcher 不记录或公开应用路径

#### Scenario：应用无法启动

- **当** 系统无法启动选中的应用
- **那么** Launcher 显示安全、可理解的失败原因
- **并且** 日志不包含用户路径或应用数据路径

### Requirement：Launcher 必须支持输入法组合态搜索

Launcher 必须在中文拼音输入法 marked text 尚未提交时，使用当前组合文本更新搜索结果。用户不得被要求确认候选、取消组合或切换为英文输入法后才能搜索应用。

#### Scenario：拼音候选未确认时搜索英文应用别名

- **假如** 当前输入法为中文拼音输入法
- **并且** 微信应用的索引包含 `WeChat` 英文别名
- **当** 用户在搜索框输入 `wechat` 且候选文本仍处于 marked 状态
- **那么** Launcher 在候选确认前显示微信应用结果
- **并且** 输入法候选窗口保持正常显示
- **并且** 搜索框不强制提交或替换输入法候选

#### Scenario：组合文本包含拼音分隔

- **当** 输入法将当前组合文本显示或报告为包含空格、撇号或宽度差异的形式
- **那么** 系统生成仅用于匹配的紧凑查询形式
- **并且** 搜索框继续显示输入法提供的原始组合文本

#### Scenario：组合期间按 Return 或方向键

- **假如** 搜索框仍存在 marked text
- **当** 用户按 Return、Up 或 Down
- **那么** 系统优先将按键交给输入法处理候选
- **并且** 不执行 Launcher 结果或移动 Launcher 选择

#### Scenario：组合期间按 Escape

- **假如** 搜索框仍存在 marked text
- **当** 用户按 Escape
- **那么** 系统优先取消输入法组合
- **并且** 不在同一次按键中关闭 Launcher

### Requirement：搜索结果必须包含文件

Launcher 必须使用 Spotlight 元数据查询提供文件结果。系统不得为实现 Launcher 建立自有全盘索引、递归扫描整个磁盘或读取文件内容。

#### Scenario：搜索 Spotlight 文件

- **当** 用户输入非空文件查询
- **并且** Spotlight 可用且存在匹配元数据
- **那么** 系统异步显示有限数量的文件结果
- **并且** 不读取文件内容

#### Scenario：打开文件

- **当** 用户执行一个当前可访问的文件结果
- **那么** 系统通过 macOS 默认打开机制打开文件
- **并且** 关闭 Launcher 面板

#### Scenario：Spotlight 不可用

- **当** Spotlight 未完成索引、不可用或系统限制当前查询
- **那么** Launcher 显示文件搜索不可用或受限原因
- **并且** 命令和应用搜索继续可用
- **并且** 系统不回退到递归全盘扫描

#### Scenario：文件不可访问

- **当** 用户执行的文件结果因 App Sandbox 或权限限制不可访问
- **那么** 系统显示资源不可访问原因
- **并且** 不把该失败误报为文件不存在
- **并且** 不请求与 Launcher 无关的高权限

### Requirement：异步搜索必须保持结果时序正确

系统必须取消过期查询，并确保旧查询结果不能覆盖较新查询。单个搜索提供者失败不得清除其他提供者已经返回的有效结果。

#### Scenario：用户快速连续输入

- **当** 用户在前一查询完成前继续修改查询
- **那么** 系统取消或废弃前一查询
- **并且** 只接受当前查询代次的结果
- **并且** 旧结果不会覆盖新结果

#### Scenario：部分搜索提供者失败

- **当** 文件提供者失败但命令或应用提供者成功
- **那么** 系统继续展示成功结果
- **并且** 显示非阻塞的文件搜索降级状态

#### Scenario：文件搜索慢于应用搜索

- **当** 命令或应用提供者已返回当前查询结果
- **并且** Spotlight 文件提供者仍在运行
- **那么** 系统立即发布并显示本地结果
- **并且** 不等待文件提供者完成后才首次更新列表
- **并且** 文件结果到达后以同一查询代次增量合并

#### Scenario：异步结果更新选择

- **当** 新一批结果到达
- **并且** 当前选中结果仍然存在
- **那么** 系统保持当前选择
- **否则** 系统选择第一个可执行结果或清除选择

#### Scenario：面板关闭时存在搜索任务

- **当** Launcher 面板关闭
- **那么** 系统取消未完成搜索
- **并且** 不继续发布结果到已关闭面板

### Requirement：搜索排序必须稳定且可预测

系统必须优先展示强匹配，并在相同分数下使用稳定次级排序，避免相同查询的结果无原因跳动。

#### Scenario：存在不同匹配强度

- **当** 同一查询产生完全匹配、前缀匹配和子串匹配
- **那么** 完全匹配排在前缀匹配之前
- **并且** 前缀匹配排在弱子串匹配之前

#### Scenario：结果重复

- **当** 同一命令、应用或文件由一个或多个批次重复返回
- **那么** 系统只展示一个规范结果
- **并且** 去重标识不进入公开日志

### Requirement：Launcher 必须保护搜索隐私

Launcher 的搜索、匹配和命令路由必须默认完全在本地完成。系统不得持久化或上传搜索词、文件名、文件路径或结果标题，也不得将这些内容写入日志。

#### Scenario：执行普通搜索

- **当** 用户搜索应用、文件或命令
- **那么** 查询文本仅用于当前进程内的搜索
- **并且** 查询文本不写入设置、数据库或日志
- **并且** 不向网络服务发送查询

#### Scenario：记录搜索错误

- **当** 某个搜索提供者发生错误
- **那么** 日志只记录提供者类型、稳定错误代码和已批准的安全上下文
- **并且** 不记录查询词、文件名、文件路径或结果标题

#### Scenario：关闭面板

- **当** 用户关闭 Launcher 面板
- **那么** 系统清理瞬态查询和结果
- **并且** 不建立搜索历史

### Requirement：Launcher 必须限制资源消耗

Launcher 不得因搜索显著增加系统负载。昂贵查询必须受 debounce、取消和结果上限约束，面板关闭后必须停止不必要工作。

#### Scenario：空查询

- **当** 查询为空
- **那么** 系统不启动 Spotlight 文件查询
- **并且** 只展示有限的本地命令或应用建议

#### Scenario：连续修改查询

- **当** 用户快速连续输入
- **那么** 系统对昂贵文件查询应用短暂 debounce
- **并且** 命令匹配不会被不必要阻塞

#### Scenario：应用索引尚未建立

- **当** 应用启动或 Launcher 首次显示
- **那么** 系统在后台预热应用索引
- **并且** 不在主线程递归扫描应用目录
- **并且** 并发查询复用同一个索引刷新任务而不重复扫描

#### Scenario：本地结果与文件结果并发搜索

- **当** 当前查询同时匹配应用和文件
- **那么** 已缓存应用结果作为首批结果发布
- **并且** Spotlight debounce 和执行不会阻塞首批本地结果

#### Scenario：预热索引满足本地搜索延迟预算

- **假如** 应用索引已经预热
- **并且** 性能基准使用固定应用样本并完成一次热身
- **当** 系统重复执行应用匹配与首批本地结果发布
- **那么** 应用匹配耗时 P95 不超过 50ms
- **并且** Launcher Store 收到首批本地结果的耗时 P95 不超过 100ms
- **并且** 该预算不包含首次索引构建与 Spotlight 系统延迟

#### Scenario：新查询取代 Spotlight 查询

- **当** 用户在旧 Spotlight 查询完成前继续输入或关闭面板
- **那么** 系统显式停止旧 `NSMetadataQuery`
- **并且** 旧查询不继续占用不必要资源或发布结果

#### Scenario：文件结果数量过多

- **当** Spotlight 返回大量匹配项
- **那么** 系统只向 UI 发布受限数量的候选
- **并且** 不在主线程处理无界结果集合
