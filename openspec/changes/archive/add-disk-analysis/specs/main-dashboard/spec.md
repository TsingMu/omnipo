# main-dashboard 增量规范

## MODIFIED Requirements

### Requirement: Dashboard 必须说明产品定位与当前状态

Dashboard MUST 展示 Omnipo 品牌说明、启动磁盘状态卡和安全说明。系统在应用启动后必须尽快以只读方式读取启动卷容量摘要，并在可用时显示已用空间、可用空间和总容量；当容量摘要不可用时，系统必须明确显示原因，不得展示虚构容量、占用率或节省空间。

#### Scenario: 首次查看 Dashboard 且容量摘要可用

- **当** 用户打开 Dashboard
- **并且** 系统已成功读取启动卷容量摘要
- **那么** 状态卡显示已用空间、可用空间和总容量
- **并且** 这些数字来自卷级只读元数据
- **并且** 不读取用户文件内容或启动目录扫描

#### Scenario: 首次查看 Dashboard 时仍在读取容量摘要

- **当** 用户打开 Dashboard
- **并且** 容量摘要尚未返回
- **那么** 状态卡显示加载中状态
- **并且** 三项容量字段使用非数字占位符
- **并且** 主窗口其余内容保持可用

#### Scenario: 容量摘要不可用

- **当** 用户打开 Dashboard
- **并且** 系统暂时无法读取启动卷容量摘要
- **那么** 状态卡显示容量信息不可用及原因
- **并且** 不展示演示容量、占用百分比或虚构节省空间
- **并且** 不回退为递归扫描整个磁盘

### Requirement: Dashboard 快捷入口必须保持纯导航

Dashboard 中的快捷入口 MUST 只负责将用户导航到对应目的地。容量摘要状态变化、刷新或降级不得让快捷入口自动触发扫描、清理、删除或其他副作用行为。

#### Scenario: 点击首页快捷入口

- **当** 用户在 Dashboard 点击“磁盘清理”或其他快捷入口
- **那么** 系统仅执行导航
- **并且** 不启动任何目录扫描、清理、删除或后台任务
