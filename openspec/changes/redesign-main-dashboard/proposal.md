# Change：重设计主窗口与 Dashboard

## 为什么需要此变更

当前主窗口只有扁平的英文导航和 Dashboard 占位页，尚未形成 macOS 管家应用应有的清晰层级。用户希望参考成熟 Mac 管家产品的布局，快速理解应用定位、当前状态与常用入口，同时保持 Omnipo 的本地优先和保守策略。

## 变更内容

- 将主窗口整理为稳定的原生侧栏与详情布局，并按概览、效率工具、系统工具分组。
- 为侧栏入口提供中文标题、简短说明和语义图标。
- 将 Dashboard 改为居中品牌介绍、启动磁盘状态卡、四个常用功能快捷入口和安全说明。
- 使用系统材质、语义色和自适应布局，支持浅色、深色与窗口缩放。
- Dashboard 快捷入口只负责导航，不自动扫描、清理、卸载、审计或读取微信数据。
- 未接入磁盘服务前明确显示“尚未扫描”，不展示虚构容量、占用率或节省空间。

## 影响范围

### 新增能力

- `main-dashboard`：主窗口信息架构、Dashboard 展示与快捷导航。

### 修改的现有能力

- `application-foundation`：主窗口仍使用既有 `NavigationSplitView` 和稳定导航状态，只调整展示结构与中文文案。

## 非目标

- 不实现磁盘扫描、清理、卸载、权限审计、微信分析或系统监控服务。
- 不读取磁盘容量、TCC 数据库、微信目录或用户文件。
- 不复刻参考产品的品牌、图标或专有视觉资产。
- 不改变 Launcher 搜索面板及其未完成的人工验收任务。

## 成功标准

- 主窗口在默认尺寸下呈现清晰的分组侧栏和 Dashboard 层级。
- Dashboard 四个快捷入口可导航到现有对应页面。
- 未扫描状态不包含虚构指标，也不会触发任何后台敏感操作。
- 布局在 720×480 最小窗口和深浅外观下保持可读、可滚动。
- 工程可编译且全部自动化测试通过。
