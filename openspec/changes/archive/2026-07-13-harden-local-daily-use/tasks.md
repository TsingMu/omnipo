# Tasks: Harden Local Daily Use

> 本 change 只加固本地长期使用，不增加发布流程、云服务、自动更新或清理删除能力。每完成一项必须更新 `[ ]` 为 `[x]`，并保持工程可编译。

- [x] 1.2 记录本地基线：macOS 版本、Xcode 版本、应用版本和全量测试结果，不提交设备名、用户路径或账号信息。

## 2. 开机启动状态一致性

- [x] 2.1 定义可注入的 `LaunchAtLoginService` 与稳定状态模型，隔离 `SMAppService`。
- [x] 2.2 实现系统服务适配器，映射 enabled、disabled、requiresApproval、notFound 等状态，不暴露原始错误内容。
- [x] 2.3 修改设置界面，以系统有效状态初始化开关；请求失败时回滚 UI，且仅持久化已确认状态。
- [x] 2.4 防止初始化加载触发无意注册，并在请求期间禁用重复操作。
- [x] 2.5 为状态映射、成功注册/注销、失败回滚和需要用户批准补充单元测试。

## 3. 剪贴板非致命降级

- [x] 3.1 为剪贴板服务增加明确的可用性表达，避免 UI 将初始化失败当成空历史。
- [x] 3.2 实现 `UnavailableClipboardService`，不启动监控并为所有依赖存储的操作返回稳定脱敏错误。
- [x] 3.3 将 `DependencyContainer.makeClipboardService` 的 `preconditionFailure` 替换为日志脱敏的 fallback 装配。
- [x] 3.4 修改剪贴板主页面与面板，在服务不可用时显示原因和安全恢复建议，不尝试自动删除或重建数据库。
- [x] 3.5 注入 Application Support、SQLite 和 schema 初始化失败，验证主应用其余能力仍可创建和使用。

## 4. 目录授权恢复

- [x] 4.1 为磁盘扫描根定义未配置、可用和需要重新授权三类状态及稳定原因码。
- [x] 4.2 修改 `AuthorizedRootManager`，刷新可恢复的 stale bookmark，并显式保留不可恢复状态供 UI 展示。
- [x] 4.3 修改磁盘分析 UI，为失效授权显示“重新选择目录”，不误报为空结果或 0 B。
- [x] 4.4 为微信授权根提供等价的失效计数/恢复状态；有效根仍可产生部分扫描结果。
- [x] 4.5 修改微信管理 UI，区分未授权、部分授权失效和无数据，并提供重新授权入口。
- [x] 4.6 覆盖有效、stale 可刷新、bookmark 损坏、scope 拒绝、重复根和部分失效测试，确保日志无路径及 bookmark 数据。
