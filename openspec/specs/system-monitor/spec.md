# system-monitor Specification

## 元信息

- **能力名称**: `system-monitor`
- **首次实现 change**: `add-system-monitor`
- **验收日期**: 2026-06-26
- **当前状态**: 已实现并合并

## Purpose

system-monitor capability 在主窗口“系统监控”页面提供 CPU、内存、能耗、磁盘与网络五个维度的整机级只读概览。能力仅使用 App Sandbox 内可用的公开 API 进行受控采样，支持 1 至 30 秒采样间隔、显式刷新、页面活跃期采样与离开即停；所有指标仅保留内存态快照，不写盘、不上传、不在日志中记录具体数值或接口名。

## Requirements
### Requirement: 系统监控页必须提供五维度只读概览

系统 MUST 在主窗口“系统监控”页面提供 CPU、内存、能耗、磁盘和网络五个维度的整机级只读概览。五维度概览 MAY 在“纵览”页以紧凑摘要展示，并在对应顶部标签页中展示专项详情。每个维度在数据可用时展示真实当前值，不可用时展示一致、可理解的降级原因；不得伪造数值、百分比或速率。

#### Scenario: 纵览页显示五维度摘要

- **当** 用户进入“系统监控”页面
- **并且** 五个维度的采样均成功
- **那么** 纵览页显示 CPU、内存、能耗、磁盘、网络五个维度的整机摘要
- **并且** 每个摘要展示对应维度的真实当前值
- **并且** 不展示虚构数字或演示百分比

#### Scenario: 专项页显示单维度详情

- **当** 用户切换到 CPU、内存、能耗、磁盘或网络标签
- **那么** 页面展示对应维度的详情
- **并且** 详情使用与纵览页同一采样快照
- **并且** 不因标签切换重复读取磁盘容量或重复启动网络采样

#### Scenario: 单维度不可用

- **当** 五维度中某一维度采样失败
- **那么** 纵览页对应摘要显示降级原因
- **并且** 对应专项页显示同一降级原因
- **并且** 其他维度的真实值继续展示
- **并且** 不以 0 或空状态伪装成功

### Requirement: 系统监控必须使用公开 API 且不申请额外权限

系统 MUST 在 App Sandbox 内完成全部采样，不得申请辅助功能、输入监控、完全磁盘访问或其他与系统资源监控无关的权限。CPU 使用 Mach `host_processor_info`；内存使用 Mach `host_statistics64` 与 `sysctl`；电池使用 `IOPSCopyPowerSourcesInfo`；网络使用 `getifaddrs`；磁盘容量复用 `DiskUsageService`。不得调用 SMC、PowerMetrics、`iostat` 子进程或其他私有/特权 API。

#### Scenario: 首次使用系统监控

- **当** 用户首次进入“系统监控”页面
- **那么** 系统不弹出任何隐私权限请求
- **并且** 不读取其他应用的状态或文件内容
- **并且** 不调用任何子进程

#### Scenario: 维度因 API 不可用降级

- **当** 某公开 API 在当前系统不可用或被 Sandbox 拒绝
- **那么** 对应卡片显示降级原因
- **并且** 不绕过 Sandbox、不申请额外权限

### Requirement: 系统监控必须支持可取消的受控采样

系统 MUST 仅在“系统监控”页面活跃时进行后台采样；用户离开页面或关闭主窗口时立即停止采样。采样必须支持取消，旧采样任务不得在新任务启动后继续推送结果。

#### Scenario: 进入页面启动采样

- **当** 用户进入“系统监控”页面
- **那么** 系统启动后台采样任务
- **并且** 立即推送一次首帧
- **并且** 后续按配置间隔定期采样

#### Scenario: 离开页面停止采样

- **当** 用户切换到其他页面或关闭主窗口
- **那么** 系统立即取消后台采样任务
- **并且** 不再推送任何快照
- **并且** Omnipo 自身 CPU 占用回落，不构成后台监控负担

#### Scenario: 快速刷新与频率切换

- **当** 用户连续点击刷新或快速切换采样间隔
- **那么** 旧采样任务被取消
- **并且** 只接受当前采样代次的结果
- **并且** 旧结果不会覆盖新结果

### Requirement: 采样间隔必须可配置且范围受限

系统 MUST 允许用户在“系统监控”页面或设置中配置采样间隔，默认值为 5 秒，可调范围为 1 至 30 秒。系统不得接受 0、负值或超过上限的间隔。

#### Scenario: 调整采样间隔

- **当** 用户将采样间隔改为合法范围内的值
- **那么** 新间隔在下次采样周期生效
- **并且** 设置在重启后保留

#### Scenario: 输入非法间隔

- **当** 用户尝试设置 0、负值或超过 30 秒的间隔
- **那么** 系统拒绝该值
- **并且** 保持上一次有效间隔
- **并且** 不破坏采样稳定性

#### Scenario: 损坏配置回退

- **当** 应用启动时读取到损坏或越界的采样间隔设置
- **那么** 系统不崩溃
- **并且** 回退到默认 5 秒
- **并且** 日志只记录稳定错误代码

### Requirement: 系统监控必须保护用户隐私

系统 MUST 保证所有采样的指标仅在内存中保留短暂快照，不写入磁盘、不上传网络、不进入 OSLog 的数值细节。日志只记录稳定事件码（启动、停止、采样失败）与维度缩写，不得记录具体数值、网络接口名、电池百分比、内存字节数或 CPU 百分比。

#### Scenario: 普通采样

- **当** 系统按间隔采样五个维度
- **那么** 指标仅用于当前 UI 展示
- **并且** 不写入磁盘或数据库
- **并且** 不向任何网络服务发送数据

#### Scenario: 记录采样事件

- **当** 系统记录采样启动、停止或失败事件
- **那么** 日志只包含稳定事件码与维度缩写
- **并且** 不包含具体数值、接口名、电池百分比等可识别信息

#### Scenario: 关闭页面

- **当** 用户离开“系统监控”页面
- **那么** 系统停止采样
- **并且** 不保留历史趋势或快照缓存（除非后续 change 显式启用）

### Requirement: 系统监控磁盘卡必须复用 Disk Usage 数据源

磁盘卡 MUST 与 Dashboard、Disk Analysis 共享同一 `DiskCapacityAvailability` 状态来源，避免重复扫描与状态分裂。本 change 不实现实时磁盘 IOPS；磁盘卡仅展示容量摘要，IOPS 不可用时不伪造数据。

#### Scenario: 磁盘卡数据一致

- **当** 用户在 Dashboard、Disk Analysis 与 System Monitor 之间切换
- **那么** 三处的启动卷容量数字一致
- **并且** 不重复触发卷元数据读取

#### Scenario: IOPS 不可用降级

- **当** 磁盘卡展示读写吞吐
- **那么** 卡片明确说明实时 IOPS 需私有 API，本 change 不实现
- **并且** 不伪造吞吐数字

### Requirement: 系统监控必须支持无障碍与外观适配

系统监控页 MUST 使用系统语义色与材质，自动适配浅色与深色外观，并为每张卡片提供 VoiceOver 标签与键盘可达性。状态差异不得仅依赖颜色。

#### Scenario: 切换浅色与深色外观

- **当** 系统外观发生变化
- **那么** 五张卡片与降级文案保持可读
- **并且** 选中态、占用态、警告态不依赖颜色差异

#### Scenario: 使用 VoiceOver 浏览

- **当** 视障用户使用 VoiceOver 浏览系统监控页
- **那么** 每张卡片有清晰的语音标签
- **并且** 占用百分比、电池状态、网络速率均可被读出
- **并且** 降级原因可被读出

### Requirement: 系统监控页必须通过顶部标签卡切换资源页面

系统 MUST 在“系统监控”页面顶部提供标签卡，用于在纵览、CPU、内存、能耗、磁盘和网络页面之间切换。标签切换 MUST 只改变当前展示内容，不得启动重复采样任务或重置采样间隔。

#### Scenario: 进入系统监控默认显示纵览

- **当** 用户进入“系统监控”页面
- **那么** 顶部显示纵览、CPU、内存、能耗、磁盘、网络标签
- **并且** 默认选中“纵览”
- **并且** 页面启动一条受控采样链路

#### Scenario: 切换到专项页

- **当** 用户点击“CPU”或“内存”等标签
- **那么** 页面切换到对应资源专项内容
- **并且** 采样间隔设置保持不变
- **并且** 不创建第二条整机指标采样任务

#### Scenario: 标签卡可访问

- **当** 用户使用键盘或 VoiceOver 浏览系统监控页
- **那么** 每个标签都有清晰名称
- **并且** 当前选中标签可被读出
- **并且** 选中态不只依赖颜色区分

### Requirement: 资源专项页必须显示对应维度的 APP 使用情况

系统 MUST 在 CPU、内存等资源专项页的详情下方显示对应维度的 APP 使用情况列表。纵览页 MUST 只展示 CPU、内存、能耗、磁盘、网络整机总体使用情况，不得在纵览页展示 APP 排行。APP 使用列表 MUST 展示当前运行应用的资源使用快照；不得展示历史使用时长、演示应用或虚构数值。

#### Scenario: CPU APP 使用情况可用

- **当** 用户进入系统监控 CPU 页
- **并且** APP 使用采样成功
- **那么** CPU 页在 CPU 详情下方显示 APP CPU 使用排行
- **并且** 记录按照 CPU 占有率从高到低排列
- **并且** 每条记录展示应用名称、应用图标或占位图标、CPU 和内存信息
- **并且** 记录来自当前采样周期

#### Scenario: 内存 APP 使用情况可用

- **当** 用户进入系统监控内存页
- **并且** APP 使用采样成功
- **那么** 内存页在内存详情下方显示 APP 内存使用排行
- **并且** 记录按照内存占用从高到低排列
- **并且** 每条记录展示应用名称、应用图标或占位图标、CPU 和内存信息
- **并且** 记录来自当前采样周期

#### Scenario: APP 使用情况为空

- **当** APP 使用采样成功
- **并且** 当前没有可归属到用户可识别 `.app` 的运行中应用
- **那么** APP 使用情况区域显示空状态
- **并且** 不展示示例应用
- **并且** 不伪造使用量

#### Scenario: APP 使用情况不可用

- **当** APP 使用采样因进程列表、资源读取或应用归属失败而不可用
- **那么** 当前资源专项页的 APP 使用情况区域显示降级原因
- **并且** CPU、内存、能耗、磁盘和网络整机摘要继续展示可用数据
- **并且** 不以 0 或空列表伪装采样成功

#### Scenario: 纵览页不展示 APP 排行

- **当** 用户进入系统监控纵览页
- **那么** 纵览页显示 CPU、内存、能耗、磁盘、网络五个整机摘要
- **并且** 不显示 APP 使用情况列表

### Requirement: APP 使用情况必须按当前资源维度降序

系统 MUST 按当前专项页资源维度降序排列 APP 使用记录。CPU 页 MUST 按 CPU 占有率降序；内存页 MUST 按内存占用降序；无法可靠归因到应用的维度 MUST 显示降级原因，不得伪造排行。所有使用量 MUST 表示当前采样周期的资源占用，不得表示历史使用时长或用户行为频率。

#### Scenario: CPU 默认排序

- **当** CPU 页 APP 使用情况列表展示多条记录
- **那么** 记录按照 CPU 占有率从高到低排列
- **并且** CPU 占有率相同或不可用时以内存占用从高到低作为次级排序
- **并且** 仍相同时按应用名称升序，保证排序稳定

#### Scenario: 内存默认排序

- **当** 内存页 APP 使用情况列表展示多条记录
- **那么** 记录按照内存占用从高到低排列
- **并且** 内存占用相同或不可用时按应用名称升序，保证排序稳定

#### Scenario: 刷新后重新排序

- **当** 用户点击刷新
- **并且** 新采样结果中当前资源维度的应用使用量发生变化
- **那么** APP 使用情况列表根据新快照重新按当前资源维度降序排列
- **并且** 旧排序不会覆盖新结果

#### Scenario: 不把使用量解释为时长

- **当** UI 展示 APP 使用情况
- **那么** 文案不得把 `usageAmount` 表述为使用时长、前台时长或历史活跃度
- **并且** 不展示未采集的历史趋势

### Requirement: APP 使用采样必须遵守系统监控隐私边界

系统 MUST 仅在系统监控页活跃时采样 APP 使用情况，并且所有 APP 使用记录仅保存在内存中的当前快照。系统 MUST NOT 读取窗口标题、文档名、网页 URL、文件内容、聊天内容或网络内容。

#### Scenario: 普通 APP 使用采样

- **当** 系统采样 APP 使用情况
- **那么** 采样仅覆盖当前运行应用的资源快照
- **并且** 不写入数据库或文件
- **并且** 不向任何网络服务发送数据

#### Scenario: 日志记录

- **当** 系统记录 APP 使用采样启动、停止或失败事件
- **那么** 日志只包含稳定事件码
- **并且** 不包含应用名称、bundle id、进程 id、CPU 百分比、内存字节数或网络速率

#### Scenario: 离开页面

- **当** 用户切换到其他页面或关闭主窗口
- **那么** 系统停止整机指标采样
- **并且** 停止 APP 使用采样
- **并且** 不保留 APP 使用历史快照
