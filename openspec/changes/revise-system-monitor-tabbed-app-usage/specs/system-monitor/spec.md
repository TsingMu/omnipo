# system-monitor 增量规范

## ADDED Requirements

### Requirement: 系统监控页必须通过顶部标签卡切换资源页面

系统 MUST 在“系统监控”页面顶部提供标签卡，用于在纵览、CPU、内存、能耗、磁盘和网络页面之间切换。标签切换 MUST 只改变当前展示内容，不得启动重复采样任务或重置采样间隔。

#### Scenario: 进入系统监控默认显示纵览

- **当** 用户进入“系统监控”页面
- **那么** 顶部显示纵览、CPU、内存、能耗、磁盘、网络标签
- **并且** 默认选中“纵览”
- **并且** 页面启动一条受控采样链路

#### Scenario: 切换到专项页

- **当** 用户点击“CPU”或“内存”等标签
- **那么** 页面切换到对应资源专项内容
- **并且** 采样间隔设置保持不变
- **并且** 不创建第二条整机指标采样任务

#### Scenario: 标签卡可访问

- **当** 用户使用键盘或 VoiceOver 浏览系统监控页
- **那么** 每个标签都有清晰名称
- **并且** 当前选中标签可被读出
- **并且** 选中态不只依赖颜色区分

### Requirement: 纵览页必须在整机摘要下方显示 APP 使用情况

系统 MUST 在“纵览”页的整机摘要下方显示 APP 使用情况列表。列表 MUST 展示当前运行应用的资源使用快照；不得展示历史使用时长、演示应用或虚构数值。

#### Scenario: APP 使用情况可用

- **当** 用户进入系统监控纵览页
- **并且** APP 使用采样成功
- **那么** 纵览页在整机摘要下方显示 APP 使用情况列表
- **并且** 每条记录展示应用名称、应用图标或占位图标、当前使用量、CPU 和内存信息
- **并且** 记录来自当前采样周期

#### Scenario: APP 使用情况为空

- **当** APP 使用采样成功
- **并且** 当前没有可归属到用户可识别 `.app` 的运行中应用
- **那么** APP 使用情况区域显示空状态
- **并且** 不展示示例应用
- **并且** 不伪造使用量

#### Scenario: APP 使用情况不可用

- **当** APP 使用采样因进程列表、资源读取或应用归属失败而不可用
- **那么** APP 使用情况区域显示降级原因
- **并且** CPU、内存、能耗、磁盘和网络整机摘要继续展示可用数据
- **并且** 不以 0 或空列表伪装采样成功

### Requirement: APP 使用情况必须默认按使用量降序

系统 MUST 默认按当前使用量降序排列 APP 使用记录。使用量 MUST 表示当前采样周期的资源占用，不得表示历史使用时长或用户行为频率。

#### Scenario: 默认排序

- **当** APP 使用情况列表展示多条记录
- **那么** 记录按照 `usageAmount` 从高到低排列
- **并且** 使用量相同或接近时以内存占用从高到低作为次级排序
- **并且** 仍相同时按应用名称升序，保证排序稳定

#### Scenario: 刷新后重新排序

- **当** 用户点击刷新
- **并且** 新采样结果中应用使用量发生变化
- **那么** APP 使用情况列表根据新快照重新按使用量降序排列
- **并且** 旧排序不会覆盖新结果

#### Scenario: 不把使用量解释为时长

- **当** UI 展示 APP 使用情况
- **那么** 文案不得把 `usageAmount` 表述为使用时长、前台时长或历史活跃度
- **并且** 不展示未采集的历史趋势

### Requirement: APP 使用采样必须遵守系统监控隐私边界

系统 MUST 仅在系统监控页活跃时采样 APP 使用情况，并且所有 APP 使用记录仅保存在内存中的当前快照。系统 MUST NOT 读取窗口标题、文档名、网页 URL、文件内容、聊天内容或网络内容。

#### Scenario: 普通 APP 使用采样

- **当** 系统采样 APP 使用情况
- **那么** 采样仅覆盖当前运行应用的资源快照
- **并且** 不写入数据库或文件
- **并且** 不向任何网络服务发送数据

#### Scenario: 日志记录

- **当** 系统记录 APP 使用采样启动、停止或失败事件
- **那么** 日志只包含稳定事件码
- **并且** 不包含应用名称、bundle id、进程 id、CPU 百分比、内存字节数或网络速率

#### Scenario: 离开页面

- **当** 用户切换到其他页面或关闭主窗口
- **那么** 系统停止整机指标采样
- **并且** 停止 APP 使用采样
- **并且** 不保留 APP 使用历史快照

## MODIFIED Requirements

### Requirement: 系统监控页必须提供五维度只读概览

系统 MUST 在主窗口“系统监控”页面提供 CPU、内存、能耗、磁盘和网络五个维度的整机级只读概览。五维度概览 MAY 在“纵览”页以紧凑摘要展示，并在对应顶部标签页中展示专项详情。每个维度在数据可用时展示真实当前值，不可用时展示一致、可理解的降级原因；不得伪造数值、百分比或速率。

#### Scenario: 纵览页显示五维度摘要

- **当** 用户进入“系统监控”页面
- **并且** 五个维度的采样均成功
- **那么** 纵览页显示 CPU、内存、能耗、磁盘、网络五个维度的整机摘要
- **并且** 每个摘要展示对应维度的真实当前值
- **并且** 不展示虚构数字或演示百分比

#### Scenario: 专项页显示单维度详情

- **当** 用户切换到 CPU、内存、能耗、磁盘或网络标签
- **那么** 页面展示对应维度的详情
- **并且** 详情使用与纵览页同一采样快照
- **并且** 不因标签切换重复读取磁盘容量或重复启动网络采样

#### Scenario: 单维度不可用

- **当** 五维度中某一维度采样失败
- **那么** 纵览页对应摘要显示降级原因
- **并且** 对应专项页显示同一降级原因
- **并且** 其他维度的真实值继续展示
- **并且** 不以 0 或空状态伪装成功
