# Change：添加 Launcher 核心能力

## 为什么需要此变更

Omnipo 已完成 Phase 0 应用基础，但 Launcher 仍是占位页面。用户目前必须先打开主窗口，再通过侧边栏进入各功能，无法像 Alfred 一样从任意应用快速发起搜索或执行 Omnipo 命令。

Launcher 是后续 Clippo、磁盘清理、卸载、权限审计、微信空间管理和系统监控的统一入口。先建立稳定的全局快捷键、搜索结果模型、搜索面板和命令路由，可以让后续能力通过增量注册接入，而无需反复修改窗口和键盘交互基础设施。

## 变更内容

- 新增可配置的全局快捷键，默认 Option + Space。
- 检测快捷键注册冲突，在失败时保留上一次有效快捷键并向用户说明原因。
- 新增键盘优先的 Launcher 搜索面板。
- 支持应用、文件和 Omnipo 功能命令三类搜索结果。
- 新增应用搜索与 Spotlight 文件搜索基础能力。
- 新增结果排序、去重、异步更新和过期查询取消机制。
- 支持方向键选择、Return 执行、Escape 关闭和鼠标点击执行。
- 新增以下内置命令：
  - 打开剪切板
  - 扫描磁盘
  - 卸载应用
  - 权限审计
  - 查看微信占用
  - 打开系统监控
- 将快捷键设置通过既有 `SettingsService` 仅保存在本地。
- 扩展依赖装配，新增 `ShortcutService`、`SearchService` 和 Launcher 命令执行边界。
- 为快捷键、搜索排序、命令路由、面板状态和隐私约束提供测试。

## 影响范围

### 新增能力

- `launcher`：提供全局快捷键、搜索面板、应用搜索、文件搜索与功能命令。

### 修改的现有能力

- `application-foundation`：依赖容器将装配 Launcher 所需服务；主窗口导航提供命令路由目标；设置窗口增加快捷键设置区。

### 后续能力

- `integrate-clippo` 将复用 Launcher 的“打开剪切板”命令入口，不在本 change 中实现剪切板历史。
- Cleaner、Uninstaller、Permission Audit、WeChat Manager 和 System Monitor 将复用既有命令标识和导航路由。

## 非目标

- 不集成 Clippo，不读取或保存剪切板历史。
- 不实现磁盘扫描、清理、卸载、权限审计、微信分析或系统监控业务逻辑。
- 不进行文件内容全文解析，不读取用户文件内容。
- 不建立自有全盘文件索引或后台遍历用户磁盘。
- 不实现网络搜索、云同步、插件市场或第三方搜索提供者。
- 不申请辅助功能、输入监控或完全磁盘访问权限。
- 不绕过 App Sandbox 或 Spotlight 的系统访问限制。
- 不引入第三方快捷键、模糊搜索或数据库依赖。

## 用户价值

- 用户可从任意前台应用通过 Option + Space 唤起 Omnipo。
- 用户可主要依靠键盘查找应用、文件和功能入口。
- 用户可在设置中更改快捷键，并在发生冲突时得到明确反馈。
- 用户可快速进入尚未实现的功能页面，为后续 change 保持稳定入口。

## 风险概述

- 全局快捷键可能与系统或其他应用冲突。
- `NSPanel` 的激活、焦点和多显示器定位需要谨慎处理。
- Spotlight 结果受索引状态、App Sandbox 和系统隐私设置影响。
- 搜索查询、文件名和路径属于敏感信息，不得进入日志。
- 快速连续输入可能造成旧查询覆盖新结果，需要查询代次和取消机制。

详细风险与降级策略见 `design.md`。

## 成功标准

- 首次运行时默认注册 Option + Space，且不要求辅助功能权限。
- 用户可以配置新的快捷键；冲突时旧快捷键继续有效。
- 快捷键可在任意普通前台应用中打开或关闭搜索面板。
- 面板打开后搜索框自动获得输入焦点。
- 空查询至少展示内置功能命令。
- 非空查询可异步展示应用、文件和功能命令结果。
- 用户可通过键盘完整完成查询、选择、执行和关闭流程。
- 六个内置命令可导航到对应现有页面。
- 文件搜索不可用时显示原因，不回退到全盘遍历。
- 日志不包含搜索词、应用用户数据、文件名或路径。
- 工程持续可编译，自动化测试通过，人工验收通过后才合并规范并归档 change。
