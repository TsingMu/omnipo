# Proposal：add-launcher-workbench

## 背景

Omnipo 已经具备可用的 Launcher 核心能力：全局快捷键、独立搜索面板、应用搜索、Spotlight 文件搜索和六个功能命令均可工作。但主窗口左侧栏中的“快速启动”详情页仍停留在占位说明阶段，用户进入该页后仍会看到“Phase 0 暂未实现”，容易误解为 Launcher 整体尚未完成。

## 问题

- 主窗口“快速启动”页面与实际 Launcher 能力不一致，增加认知落差。
- 用户即使已经进入“快速启动”页，仍需要再点击按钮或使用快捷键才能接触真实搜索工作流。
- 当前占位页无法体现 Launcher 已有的搜索、结果选择与结果执行能力。

## 目标

- 将主窗口“快速启动”页升级为正式 Launcher 工作台，直接展示搜索框、结果列表和执行反馈。
- 复用现有 `LauncherStore`、搜索服务、结果执行器与应用图标缓存，不创建第二套语义不一致的 Launcher。
- 保留独立浮动面板能力，同时让主窗口页成为一个可持续使用的桌面工作台视图。

## 非目标

- 不重做全局快捷键、NSPanel 生命周期或 Spotlight 搜索后端。
- 不在本 change 中增加新的 Launcher 命令、结果类型或排序规则。
- 不将主窗口页改造成与浮动面板像素级一致的视觉复制品，只要求交互能力一致。
- 不实现剪切板历史、目录扫描或其他仍未完成的业务能力。

## 受影响能力

- `launcher`：新增主窗口内嵌 Launcher 工作台要求。

## 成功标准

- 用户打开主窗口“快速启动”页时，可直接输入查询并看到命令、应用与文件结果。
- 主窗口页不再显示“Phase 0 暂未实现”占位说明。
- 用户可在该页通过键盘或鼠标执行结果，行为与独立 Launcher 面板保持一致。
- 页面内结果执行失败时，继续使用现有安全错误反馈，不暴露路径等敏感信息。
