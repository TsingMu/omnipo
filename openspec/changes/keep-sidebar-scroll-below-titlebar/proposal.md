# Change：固定侧栏滚动视口于标题栏下方

## 为什么需要此变更

上一版使用 `List.contentMargins` 给侧栏滚动内容增加顶部安全区。该 margin 属于可滚动内容的一部分；当用户选择“快速启动”或“剪切板”时，系统为显示选中行自动调整滚动位置，顶部 margin 会随内容滚出视口，导致“总览”再次进入标题栏并与窗口按钮重叠。

## 变更内容

- 将标题栏安全区从 List 的滚动内容 margin 改为 List 外部的固定视口占位。
- 当 SwiftUI 侧栏安全区小于统一工具栏真实高度时，通过只读的窄 AppKit 边界读取窗口内容布局高度。
- 侧栏列表只在标题栏下方区域滚动，任何选择或自动滚动均不能越过该边界。
- 保持系统侧栏材质、原生选中态、分组、键盘导航和窗口缩放行为。
- 使用现有安全区布局计算，并补充针对固定视口语义的测试命名。

## 影响范围

### 修改的现有能力

- `main-dashboard`：强化侧栏与标题栏的永久空间隔离。

## 非目标

- 不调整详情页占位内容或 Dashboard 视觉。
- 不写死标题栏高度，不移动或修改窗口控制按钮。
- 不修改任何扫描、删除、权限或微信功能。

## 成功标准

- 默认显示、切换到快速启动、切换到剪切板时，侧栏首行均不进入标题栏。
- 自动滚动和手动滚动不能将任何侧栏行滚到窗口控制按钮下方。
- 工程可编译且全部自动化测试通过。
