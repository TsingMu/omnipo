# disk-analysis Specification

## 元信息

- **能力名称**:disk-analysis
- **首次实现 change**:`add-disk-analysis`
- **验收日期**:2026-06-24
- **当前状态**:已实现并合并

## Purpose

disk-analysis capability 提供启动卷容量摘要与大文件列表的只读视图。容量摘要来自系统卷级元数据;大文件列表由用户通过 `NSOpenPanel` 主动授权扫描根目录后,基于 security-scoped bookmark 持久化,扫描只读 `fileSize`/`contentModificationDate`/`isRegularFile` 元数据,不读取文件内容、不建立自有索引、不执行删除或清理动作。
## Requirements
### Requirement: 磁盘分析页必须提供只读容量概览

磁盘分析页 MUST 展示当前启动卷或主工作卷的只读容量摘要，包括已用空间、可用空间和总容量。此能力只允许读取卷级元数据，不得通过递归遍历目录、建立自有全盘索引或读取文件内容来生成首页或详情页的容量数字。

#### Scenario: 容量摘要可用

- **当** 用户打开磁盘分析页
- **并且** 系统已成功读取目标卷容量摘要
- **那么** 页面显示已用空间、可用空间和总容量
- **并且** 与 Dashboard 状态卡展示相同来源的数据

#### Scenario: 容量摘要不可用

- **当** 用户打开磁盘分析页
- **并且** 系统暂时无法读取目标卷容量摘要
- **那么** 页面显示容量信息不可用及原因
- **并且** 不以 0 B 或空状态伪装成功结果

### Requirement: 磁盘分析页必须允许显式刷新

磁盘分析页 MUST 为卷容量和授权目录分析提供相互独立、用户可感知的显式刷新入口。容量刷新只能重新读取卷级容量摘要；目录分析刷新只能重新读取当前授权根中的文件系统元数据。筛选、排序、选择和忽略操作不得触发任何刷新。

#### Scenario: 用户刷新容量摘要

- **当** 用户触发容量刷新
- **那么** 系统只重新读取目标卷容量摘要
- **并且** Dashboard 与磁盘分析页共享更新后的容量状态
- **并且** 不开始目录扫描或改变当前 workbench 候选状态

#### Scenario: 用户刷新目录分析

- **当** 用户触发目录分析刷新
- **那么** 系统只重新扫描当前授权根的只读文件元数据
- **并且** 取消或废弃仍在进行的旧扫描
- **并且** 新结果替换后清空会话内选择与忽略状态
- **并且** 不执行删除、移动、内容读取或容量摘要刷新

### Requirement: 磁盘分析页必须展示大文件列表

磁盘分析页 MUST 提供大文件展示区块，并按文件大小从大到小排序结果。每条结果至少展示文件名、路径和大小等只读元信息；系统不得为此读取文件内容。在 App Sandbox 下，默认用户目录不可读，必须由用户通过 `NSOpenPanel` 显式授权扫描根目录;授权以 security-scoped bookmark 形式持久化。

#### Scenario: 大文件结果可用

- **当** 用户已授权扫描目录
- **并且** 系统已成功生成大文件结果
- **那么** 页面按大小降序展示文件列表
- **并且** 每条结果包含文件名、路径和大小
- **并且** 不展示文件内容预览

#### Scenario: 大文件结果不可用

- **当** 用户尚未授权扫描目录,或扫描失败
- **那么** 页面显示大文件列表不可用及原因
- **并且** 不伪造示例文件结果

#### Scenario: 跨重启保留授权

- **当** 用户重启应用
- **并且** 之前已授权扫描目录
- **那么** 系统从持久化的 security-scoped bookmark 还原授权
- **并且** 不要求用户再次选择

### Requirement: 大文件刷新必须保持保守边界

大文件刷新 MUST 只读取文件系统元数据并支持结果条数限制与旧任务取消。系统不得在刷新过程中执行删除、移动到废纸篓、文件内容解析或自有持久索引构建。

#### Scenario: 用户刷新大文件列表

- **当** 用户在磁盘分析页触发刷新
- **那么** 系统重新读取大文件元数据
- **并且** 结果数量被限制在预设上限内(默认 50)
- **并且** 若上一次刷新仍在进行，则旧任务被取消或废弃
- **并且** 不执行删除或内容读取

#### Scenario: 扫描系统子树

- **当** 授权目录下包含 `Library`、`.Trash`、`Caches`、`Containers`、`node_modules`、`.git` 等系统/缓存子树
- **那么** 扫描跳过这些子树及其全部后代
- **并且** 不把系统缓存或开发依赖展示为用户的大文件

### Requirement: 磁盘分析页必须清晰标注未实现能力

磁盘分析页 MUST 准确区分已实现的容量摘要、授权目录扫描和只读分析能力，与仍未实现的清理执行能力。页面不得继续把已经实现的目录扫描或分类工作台标记为 Phase 0 占位功能。

#### Scenario: 查看磁盘分析能力说明

- **当** 用户查看磁盘分析页
- **那么** 页面说明容量摘要与授权目录只读分析已经可用
- **并且** 说明当前结果受授权根和数量上限约束
- **并且** 明确标注删除、移动到废纸篓和自动清理尚未提供
- **并且** 页面不显示“目录分析尚未实现”或等价的过时文案

### Requirement: 磁盘分析必须保持保守权限边界

磁盘分析 capability MUST 在无额外隐私权限前提下工作。系统不得为了展示容量摘要申请完全磁盘访问、辅助功能、输入监控或其他与卷级容量读取无关的权限。大文件扫描需要的目录访问权限只能由用户主动通过 `NSOpenPanel` 授权,且范围严格限制在所选目录。

#### Scenario: 首次使用磁盘分析

- **当** 用户首次打开 Dashboard 或磁盘分析页
- **那么** 系统不请求额外隐私权限(容量摘要)
- **并且** 不读取文件内容
- **并且** 不产生删除、副作用写入或后台常驻扫描

#### Scenario: 用户撤销授权

- **当** 用户点击"清除"释放大文件扫描授权
- **那么** 系统释放 security scope 并删除持久化 bookmark
- **并且** 不再扫描该目录

### Requirement: 持久化扫描授权失效时必须提供恢复状态

Disk Analysis MUST 区分用户从未授权扫描目录和已保存授权无法恢复。可刷新 stale bookmark 时必须安全刷新；无法解析或无法启动 security scope 时必须显示需要重新授权，而不是将结果报告为空目录、0 B 或首次未授权。

#### Scenario: stale bookmark 可以刷新

- **当** 已保存的大文件扫描目录 bookmark 已 stale
- **并且** 系统仍允许解析并访问该目录
- **那么** 应用刷新 bookmark 并继续只读元数据扫描
- **并且** 不要求用户重复选择目录

#### Scenario: 已保存授权无法恢复

- **当** 已保存 bookmark 损坏、目录移动或 security scope 无法启动
- **那么** 磁盘分析页显示需要重新授权
- **并且** 提供重新选择目录的操作
- **并且** 不把失败状态展示为 0 B、无大文件或从未授权
- **并且** 日志不包含 bookmark 数据或原始路径

### Requirement: 磁盘分析必须提供元数据派生的只读工作台

磁盘分析页 MUST 将当前用户授权、数量受限的大文件扫描结果呈现为只读分析工作台。分类和汇总只能使用文件名扩展名、大小、修改时间、授权根相对目录等文件系统元数据，不得读取或推断文件内容。

#### Scenario: 查看当前扫描结果摘要

- **当** 用户授权的目录扫描成功并返回当前结果
- **那么** 页面显示当前结果的文件数量和合计字节
- **并且** 显示元数据派生的文件类型、大小、修改时间和目录分组
- **并且** 明确说明汇总只覆盖当前授权目录和受数量上限约束的结果
- **并且** 不把扩展名分类描述为内容验证

#### Scenario: 元数据不完整

- **当** 记录缺少扩展名、修改时间或安全的相对目录信息
- **那么** 对应 facet 使用“其他”“未知”或通用目录等保守值
- **并且** 不打开文件来补全分类

### Requirement: 当前结果必须支持本地筛选与排序

磁盘分析工作台 MUST 允许用户按查询文本、元数据类型、大小、修改时间和目录组合筛选当前结果，并提供稳定的排序选项。筛选和排序只能作用于内存中的当前结果，不得启动扫描或修改文件。

#### Scenario: 组合筛选当前结果

- **当** 用户设置一个或多个筛选条件
- **那么** 页面只显示同时满足条件且未被忽略的当前记录
- **并且** 可见数量和可见字节随结果更新
- **并且** 不触发目录扫描、文件读取、日志记录或设置持久化

#### Scenario: 筛选后没有匹配

- **当** 当前扫描存在记录但没有记录满足筛选条件
- **那么** 页面显示“当前筛选无匹配”状态
- **并且** 不把该状态描述为目录为空、扫描失败或权限不足
- **并且** 提供清除筛选的操作

#### Scenario: 稳定排序

- **当** 用户选择按大小、名称或修改时间排序
- **那么** 结果使用明确的次级排序键保持稳定顺序
- **并且** 修改时间未知的记录使用固定、可测试的位置

### Requirement: 当前结果必须支持会话内候选复核

磁盘分析工作台 MUST 支持选择、忽略、复核和恢复当前结果中的候选，并显示已选数量与估算字节。候选状态只存在于内存，不能导致任何文件操作或路径持久化。

#### Scenario: 选择只读候选

- **当** 用户选择一个或多个当前记录
- **那么** 页面显示已选数量和合计字节
- **并且** 用户可以单独复核已选记录
- **并且** 不显示删除、移动、重命名或自动清理动作

#### Scenario: 忽略与恢复候选

- **当** 用户忽略一个当前记录
- **那么** 该记录从主结果中隐藏并从选择集合移除
- **并且** 用户可以在当前结果的 review 区域恢复它
- **并且** 忽略状态不写入设置、数据库、日志或文件扩展属性

#### Scenario: 扫描结果被替换

- **当** 用户刷新目录扫描并替换当前结果
- **那么** 之前的选择和忽略状态被清空
- **并且** 应用不通过持久化路径或路径哈希恢复候选身份

### Requirement: 当前大文件记录必须支持安全的 Finder 定位

磁盘分析工作台 MUST 为仍属于当前结果且仍位于当前授权根内的记录提供“在 Finder 中显示”操作。操作必须是有限时长、只读且在所有终态释放 security scope。

#### Scenario: 在 Finder 中显示当前记录

- **当** 用户请求显示一个仍存在且位于当前授权根内的记录
- **那么** Finder 选择该文件
- **并且** 应用不打开或修改文件内容
- **并且** 操作结束后释放 security scope

#### Scenario: 文件已移动或删除

- **当** 用户请求显示的当前记录已经不存在
- **那么** 应用显示稳定、脱敏的不可用提示
- **并且** 不崩溃、不重新扫描、不记录原始路径或文件名
- **并且** 释放任何已获得的 security scope

#### Scenario: 记录超出授权根

- **当** reveal 请求的路径不再位于当前授权根内或授权无法恢复
- **那么** 应用拒绝 Finder 操作并提示重新授权或刷新
- **并且** 不扩大访问范围或绕过 macOS 权限机制
- **并且** 不记录路径、文件名或 bookmark 数据

### Requirement: 磁盘工作台状态必须保持本地和非持久

查询文本、筛选条件中的目录名称、文件选择、忽略集合、Finder 目标和当前文件清单 MUST remain in memory only and MUST NOT be written to logs, settings, databases, telemetry, or network services.

#### Scenario: 用户操作只读工作台

- **当** 用户搜索、筛选、排序、选择、忽略、恢复或定位文件
- **那么** 日志最多记录稳定操作名、聚合计数和稳定结果码
- **并且** 不记录查询文本、路径、文件名、目录 facet、bookmark 数据或文件清单
- **并且** 应用不产生网络请求或遥测事件
