# Change：稳定主窗口全部导航与详情布局

## 为什么需要此变更

主窗口侧栏当前使用 `List(selection:)`。用户点击不同目的地时，系统会主动调整 List 的滚动偏移以展示选中行，即使所有入口原本都可见，也会导致整组导航上下位移。与此同时，Dashboard 之外的七个功能页共用 `PlaceholderFeatureView`，其顶部内容允许延伸到统一工具栏下方，造成图标、标题或说明被裁切。

此前针对单一截图增加安全区只能约束标题栏重叠，不能消除 List 的选择驱动滚动，也没有统一约束所有详情页。因此需要在根布局层一次性稳定全部八个目的地。

## 变更内容

- 将侧栏从会自动滚动选中项的 `List(selection:)` 改为系统侧栏材质上的稳定 `ScrollView + LazyVStack`。
- 保留三个分组、八个入口、选中高亮、辅助功能标识、键盘方向导航和窄窗口滚动。
- 将真实窗口标题栏高度提升为 `RootView` 单一布局状态，同时提供给侧栏和详情区。
- 在根详情容器统一建立标题栏下方的固定内容视口，使 Dashboard 和七个功能页均不进入工具栏区域。
- 检查总览、快速启动、剪切板、磁盘清理、应用卸载、权限审计、微信管理和系统监控全部页面。

## 影响范围

### 修改的现有能力

- `main-dashboard`：稳定主窗口侧栏与全部详情页的空间布局。
- `application-foundation`：主窗口仍使用 `NavigationSplitView`，但侧栏内容容器和详情安全区改为显式稳定布局。

## 非目标

- 不实现各功能页业务能力。
- 不改变 Launcher 面板、设置持久化、命令映射或导航目的地。
- 不触发扫描、删除、权限读取或微信数据访问。
- 不修改窗口按钮位置或写死标题栏高度。

## 成功标准

- 连续点击八个侧栏入口时，侧栏分组和未滚动内容保持固定位置。
- 所有详情页的图标、标题、说明和卡片均完整位于工具栏下方。
- 小窗口下侧栏可手动滚动，方向键可切换目的地。
- 工程可编译且全部自动化测试通过。
