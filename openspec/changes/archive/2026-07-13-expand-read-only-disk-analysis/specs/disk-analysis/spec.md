# disk-analysis 增量规范

## ADDED Requirements

### Requirement: 磁盘分析必须提供元数据派生的只读工作台

磁盘分析页 MUST 将当前用户授权、数量受限的大文件扫描结果呈现为只读分析工作台。分类和汇总只能使用文件名扩展名、大小、修改时间、授权根相对目录等文件系统元数据，不得读取或推断文件内容。

#### Scenario: 查看当前扫描结果摘要

- **当** 用户授权的目录扫描成功并返回当前结果
- **那么** 页面显示当前结果的文件数量和合计字节
- **并且** 显示元数据派生的文件类型、大小、修改时间和目录分组
- **并且** 明确说明汇总只覆盖当前授权目录和受数量上限约束的结果
- **并且** 不把扩展名分类描述为内容验证

#### Scenario: 元数据不完整

- **当** 记录缺少扩展名、修改时间或安全的相对目录信息
- **那么** 对应 facet 使用“其他”“未知”或通用目录等保守值
- **并且** 不打开文件来补全分类

### Requirement: 当前结果必须支持本地筛选与排序

磁盘分析工作台 MUST 允许用户按查询文本、元数据类型、大小、修改时间和目录组合筛选当前结果，并提供稳定的排序选项。筛选和排序只能作用于内存中的当前结果，不得启动扫描或修改文件。

#### Scenario: 组合筛选当前结果

- **当** 用户设置一个或多个筛选条件
- **那么** 页面只显示同时满足条件且未被忽略的当前记录
- **并且** 可见数量和可见字节随结果更新
- **并且** 不触发目录扫描、文件读取、日志记录或设置持久化

#### Scenario: 筛选后没有匹配

- **当** 当前扫描存在记录但没有记录满足筛选条件
- **那么** 页面显示“当前筛选无匹配”状态
- **并且** 不把该状态描述为目录为空、扫描失败或权限不足
- **并且** 提供清除筛选的操作

#### Scenario: 稳定排序

- **当** 用户选择按大小、名称或修改时间排序
- **那么** 结果使用明确的次级排序键保持稳定顺序
- **并且** 修改时间未知的记录使用固定、可测试的位置

### Requirement: 当前结果必须支持会话内候选复核

磁盘分析工作台 MUST 支持选择、忽略、复核和恢复当前结果中的候选，并显示已选数量与估算字节。候选状态只存在于内存，不能导致任何文件操作或路径持久化。

#### Scenario: 选择只读候选

- **当** 用户选择一个或多个当前记录
- **那么** 页面显示已选数量和合计字节
- **并且** 用户可以单独复核已选记录
- **并且** 不显示删除、移动、重命名或自动清理动作

#### Scenario: 忽略与恢复候选

- **当** 用户忽略一个当前记录
- **那么** 该记录从主结果中隐藏并从选择集合移除
- **并且** 用户可以在当前结果的 review 区域恢复它
- **并且** 忽略状态不写入设置、数据库、日志或文件扩展属性

#### Scenario: 扫描结果被替换

- **当** 用户刷新目录扫描并替换当前结果
- **那么** 之前的选择和忽略状态被清空
- **并且** 应用不通过持久化路径或路径哈希恢复候选身份

### Requirement: 当前大文件记录必须支持安全的 Finder 定位

磁盘分析工作台 MUST 为仍属于当前结果且仍位于当前授权根内的记录提供“在 Finder 中显示”操作。操作必须是有限时长、只读且在所有终态释放 security scope。

#### Scenario: 在 Finder 中显示当前记录

- **当** 用户请求显示一个仍存在且位于当前授权根内的记录
- **那么** Finder 选择该文件
- **并且** 应用不打开或修改文件内容
- **并且** 操作结束后释放 security scope

#### Scenario: 文件已移动或删除

- **当** 用户请求显示的当前记录已经不存在
- **那么** 应用显示稳定、脱敏的不可用提示
- **并且** 不崩溃、不重新扫描、不记录原始路径或文件名
- **并且** 释放任何已获得的 security scope

#### Scenario: 记录超出授权根

- **当** reveal 请求的路径不再位于当前授权根内或授权无法恢复
- **那么** 应用拒绝 Finder 操作并提示重新授权或刷新
- **并且** 不扩大访问范围或绕过 macOS 权限机制
- **并且** 不记录路径、文件名或 bookmark 数据

### Requirement: 磁盘工作台状态必须保持本地和非持久

查询文本、筛选条件中的目录名称、文件选择、忽略集合、Finder 目标和当前文件清单 MUST remain in memory only and MUST NOT be written to logs, settings, databases, telemetry, or network services.

#### Scenario: 用户操作只读工作台

- **当** 用户搜索、筛选、排序、选择、忽略、恢复或定位文件
- **那么** 日志最多记录稳定操作名、聚合计数和稳定结果码
- **并且** 不记录查询文本、路径、文件名、目录 facet、bookmark 数据或文件清单
- **并且** 应用不产生网络请求或遥测事件

## MODIFIED Requirements

### Requirement: 磁盘分析页必须允许显式刷新

磁盘分析页 MUST 为卷容量和授权目录分析提供相互独立、用户可感知的显式刷新入口。容量刷新只能重新读取卷级容量摘要；目录分析刷新只能重新读取当前授权根中的文件系统元数据。筛选、排序、选择和忽略操作不得触发任何刷新。

#### Scenario: 用户刷新容量摘要

- **当** 用户触发容量刷新
- **那么** 系统只重新读取目标卷容量摘要
- **并且** Dashboard 与磁盘分析页共享更新后的容量状态
- **并且** 不开始目录扫描或改变当前 workbench 候选状态

#### Scenario: 用户刷新目录分析

- **当** 用户触发目录分析刷新
- **那么** 系统只重新扫描当前授权根的只读文件元数据
- **并且** 取消或废弃仍在进行的旧扫描
- **并且** 新结果替换后清空会话内选择与忽略状态
- **并且** 不执行删除、移动、内容读取或容量摘要刷新

### Requirement: 磁盘分析页必须清晰标注未实现能力

磁盘分析页 MUST 准确区分已实现的容量摘要、授权目录扫描和只读分析能力，与仍未实现的清理执行能力。页面不得继续把已经实现的目录扫描或分类工作台标记为 Phase 0 占位功能。

#### Scenario: 查看磁盘分析能力说明

- **当** 用户查看磁盘分析页
- **那么** 页面说明容量摘要与授权目录只读分析已经可用
- **并且** 说明当前结果受授权根和数量上限约束
- **并且** 明确标注删除、移动到废纸篓和自动清理尚未提供
- **并且** 页面不显示“目录分析尚未实现”或等价的过时文案
