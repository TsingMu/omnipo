# application-foundation 增量规范

## MODIFIED Requirements

### Requirement: 应用必须提供原生 macOS 主窗口

系统 MUST 以 SwiftUI Scene 和 `NavigationSplitView` 管理可调整尺寸的主窗口。窗口必须依据当前实际内容布局建立标题栏安全边界；导航选择变化不得改变无关区域的空间位置，也不得在启动阶段执行敏感业务操作。

#### Scenario: 首次启动应用

- **当** 用户启动 Omnipo
- **那么** 系统显示主窗口
- **并且** 默认选择 Dashboard，或按明确的本地恢复设置显示上次目的地
- **并且** 启动过程不请求额外隐私权限

#### Scenario: 调整窗口尺寸

- **当** 用户调整主窗口尺寸
- **那么** 侧边栏和详情区域保持可用
- **并且** 内容不得因固定的移动端布局而不可访问

#### Scenario: 切换主窗口目的地

- **当** 用户通过侧栏、Launcher 命令或恢复状态切换目的地
- **那么** 主窗口保留相同的侧栏与详情内容原点
- **并且** 选择变化不会造成导航整体跳动或详情顶部裁切
- **并且** 不触发其他功能的后台任务
