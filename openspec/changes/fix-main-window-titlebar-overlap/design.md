# Design：主窗口标题栏安全区修复

## 问题分析

主窗口使用 `NavigationSplitView` 和统一窗口工具栏。macOS 允许侧栏材质延伸至标题栏，但侧栏 `List` 在恢复选择并自动滚动时可能把首个分组放到工具栏覆盖区。该现象在标题栏较高或系统视觉样式变化时更明显，因此固定添加 52 点或 64 点边距并不可靠。

## 方案

在 `SidebarView` 外层使用 `GeometryReader` 读取当前列的 `safeAreaInsets.top`，通过 `List.contentMargins(_:_:for:)` 将该值应用到滚动内容顶部。额外间距只影响列表内容，不给侧栏材质增加不透明遮挡，也不改变列表滚动范围和系统选中态。

布局计算抽成无副作用的 `SidebarLayout.contentTopMargin(safeAreaTop:)`：

- 安全区为正数时，使用实际安全区高度。
- 安全区为零或异常负值时，返回零。

这样可以用单元测试验证边界，同时避免引入 AppKit 窗口桥接。

## 保持不变

- `RootView` 继续拥有唯一导航选择状态。
- `SidebarView` 继续使用原生 `.sidebar` List。
- 主窗口工具栏、Launcher 按钮和目的地恢复逻辑不变。
- 不触发扫描、删除、权限或微信数据访问。

## 风险与验证

- 某些系统版本可能已经为列表提供部分顶部 inset；使用实际安全区作为滚动内容 margin，需要在真实应用窗口中确认不会产生重复留白。
- 验证默认目的地和恢复到“快速启动”两种状态，并检查窗口缩放后首行仍位于工具栏下方。
