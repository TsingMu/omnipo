# main-dashboard 增量规范

## MODIFIED Requirements

### Requirement: 主窗口必须提供清晰的分组导航

系统 MUST 在原生 macOS 侧栏中保留八个稳定功能入口，并按概览、效率工具和系统工具分组展示中文标题、语义图标与简短说明。系统必须将侧栏的可滚动视口及分组标题限制在窗口标题栏安全区域下方。标题栏避让区域不得作为可随列表滚动消失的内容 margin；当 SwiftUI 安全区不足以表达统一工具栏高度时，系统必须依据窗口实际内容布局自适应，且不得写死标题栏高度。

#### Scenario: 选择侧栏入口

- **当** 用户选择任一侧栏入口
- **那么** 详情区显示对应现有页面
- **并且** 选择状态继续支持本地恢复和 Launcher 命令导航
- **并且** 选择入口本身不启动敏感业务操作

#### Scenario: 切换到非首个侧栏入口

- **当** 用户选择快速启动、剪切板或其他非首个入口
- **那么** 系统可以自动滚动列表以显示选中行
- **并且** 顶部“总览”行不得进入标题栏或窗口控制按钮区域
- **并且** 标题栏安全区不会随滚动内容移出视口

#### Scenario: 连续切换入口

- **当** 用户在多个侧栏入口之间连续切换
- **那么** 所有侧栏行的文字和点击区域始终位于标题栏下方
- **并且** 原生选择、键盘导航和滚动行为保持可用
