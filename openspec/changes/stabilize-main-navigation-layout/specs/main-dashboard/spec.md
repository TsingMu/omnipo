# main-dashboard 增量规范

## MODIFIED Requirements

### Requirement：主窗口全部导航和详情布局必须保持稳定

系统必须为八个稳定目的地提供位置稳定的分组侧栏，并为 Dashboard 和全部功能详情页提供统一的标题栏安全边界。选择变化不得导致原本可见的侧栏内容整体上下位移，任何详情内容不得进入窗口工具栏区域。

#### Scenario：连续点击全部入口

- **当** 用户依次点击总览、快速启动、剪切板、磁盘清理、应用卸载、权限审计、微信管理和系统监控
- **那么** 未发生手动滚动时侧栏分组与入口保持固定坐标
- **并且** 只有选中高亮和详情内容发生变化
- **并且** 当前入口仍具有明确的选中视觉和辅助功能状态

#### Scenario：显示任一详情页

- **当** 主窗口显示任一稳定目的地
- **那么** 页面标题、图标、说明、卡片和操作均完整位于工具栏下方
- **并且** 页面内容不会被标题栏裁切

#### Scenario：使用键盘或小窗口导航

- **当** 用户使用方向键切换入口，或窗口高度不足以显示全部入口
- **那么** 选择按稳定目的地顺序移动并在首尾停止
- **并且** 侧栏可以滚动访问全部入口
- **并且** 仅键盘导航到不可见目标时允许按需滚动
